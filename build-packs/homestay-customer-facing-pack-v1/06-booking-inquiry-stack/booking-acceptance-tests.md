# Booking Acceptance Tests

## Inquiry Quality Test

A WhatsApp inquiry generated from the site includes:

- date
- pax
- purpose
- package if selected
- cars if relevant
- questions if provided

## Event Safety Test

A guest planning an event is forced to declare:

- event type
- estimated pax
- day use or overnight
- cars
- setup needs

## Rate Clarity Test

Before sending inquiry, the guest sees:

- rate display or rate factors
- deposit note
- final rate confirmation note

## Terms Visibility Test

Before payment or confirmation, the guest can access:

- booking terms
- house rules
- cancellation/deposit summary
- privacy notice

## No Premature Automation Test

The MVP does not require:

- login
- full payment
- live calendar
- guest account

to send a useful booking inquiry.
