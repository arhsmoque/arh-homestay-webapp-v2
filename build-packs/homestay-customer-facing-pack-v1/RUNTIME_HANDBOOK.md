# Runtime Handbook — Customer-Facing Homestay Webapp

## Use this when debugging or reviewing the public site.

### 1. Product Identity

This is a **Homestay Guest Decision Engine**, not a brochure.

### 2. Public Spine

```text
Desire → Fit → Proof → Logistics → Occasion → Trust → Action
```

### 3. Above-Fold Must Answer

- What is this?
- Where is it?
- Who is it for?
- Main hook?
- Next action?

### 4. Must Show Before Inquiry

- private pool proof
- pax / sleeping summary
- parking count
- layout / compound map
- event suitability
- location / map
- rules summary
- package/rate factors

### 5. CTA Rules

Use specific labels:

- Check availability
- View layout
- Open map
- Ask event package
- Send WhatsApp inquiry

Avoid vague labels:

- Learn more
- Explore
- More info
- Click here

### 6. Booking Rule

WhatsApp-first, purpose-specific, structured.

### 7. Content Rule

Proof before adjective.

### 8. Tech Rule

Mostly static public site. Interactive only where needed.

### 9. MVP Rejects

- F&B menu metaphor
- heavy public SPA
- login-first booking
- payment before owner confirmation
- vague DM-only pricing
- hidden rules
- full social feed embeds
- autoplay video everywhere
