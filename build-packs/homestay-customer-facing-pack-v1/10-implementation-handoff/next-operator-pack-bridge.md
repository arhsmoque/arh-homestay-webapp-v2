# Bridge to Operator-Facing Pack

The customer-facing pack intentionally stops at the public guest surface.

The next pack should be:

```text
Operator-Facing Homestay Control System Pack
```

## Operator Pack Scope

- Owner control panel
- Developer console
- Role permissions
- Booking records
- Calendar state
- Rates and packages
- Demand/rate engine scaffold
- Sales analytics
- Customer records
- Legal/privacy/rules adapter
- Audit log
- Handover governance
- Backup/recovery

## Recommended 8 Turns

1. Operator product frame, role separation, control boundaries, invariant core.
2. Role, authority, permission model.
3. Owner control panel information architecture.
4. Booking and operations model.
5. Rates, packages, and demand/rule engine.
6. Data, analytics, and observability.
7. Legal, privacy, rules, and compliance surface.
8. Technical architecture and handover pack.

## Bridge Sentence

```text
The public site should make the guest confident enough to inquire.
The operator system should make the owner confident enough to run the business.
```
