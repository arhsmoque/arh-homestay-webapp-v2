# ARH Alignment Notes

## Art of Design Alignment

```text
Purpose before components.
Finite constraints before fantasy architecture.
Adapter boundaries before feature sprawl.
Observability before silent failure.
Audit before trust collapse.
```

## AODP Alignment

```text
Surface type declared.
Risk class declared.
Names inferable.
Mutations pass verify → prepare → approve → execute.
Errors include remediation.
Evidence supports replay/recovery.
```

## Art of Agentic Naming Alignment

```text
Names are inference controls.
Component/route/schema names should reveal purpose, ownership, safety, and lifecycle.
Avoid Dashboard, Manager, DataView, InfoCard, ActionBox.
```

## Frontend Gold Alignment

```text
Design is translation.
Cards represent real objects, not decoration.
Accessibility, state coverage, responsiveness, and performance are production floor.
```
