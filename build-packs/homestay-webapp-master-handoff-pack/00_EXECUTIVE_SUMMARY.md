# Executive Summary

## Product Frame

The project should not be treated as a generic homestay website or an Airbnb clone.

It should be treated as two connected systems:

```text
1. Homestay Guest Decision Engine
2. Homestay Operator Control System
```

## Customer-Facing Purpose

Help guests answer:

```text
Is this place suitable for my family, event, retreat, or stay?
Can my group fit?
Is the pool private?
Where do we sleep, park, gather, eat, and move?
What rules apply?
Where is the place?
How do I check availability?
```

Then convert that confidence into a structured WhatsApp inquiry.

## Operator-Facing Purpose

Help the owner run the homestay after handover without:

```text
breaking the public site
losing booking truth
exposing guest data
silently changing rates or rules
depending on developer memory
```

## Master Rule

```text
Customer-facing system:
Make the guest confident enough to inquire.

Operator-facing system:
Make the owner confident enough to operate.

Developer console:
Make the maintainer confident enough to diagnose and recover.

Audit:
Make the future reviewer confident enough to know what happened.
```
