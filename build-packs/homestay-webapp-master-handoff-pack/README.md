# Homestay Webapp + Operator Console Master Handoff Pack

This bundle combines two completed design streams:

```text
01_CUSTOMER_FACING_PACK/
  Guest discovery, trust, page map, booking inquiry, content system, public stack.

02_OPERATOR_FACING_PACK/
  Owner control panel, developer console, permissions, booking state machine, rates,
  analytics, legal/privacy, audit, recovery, and handover.
```

## Core Product Split

```text
Customer-facing side:
Homestay Guest Decision Engine

Operator-facing side:
Homestay Operator Control System
```

## Core Doctrine

```text
Public site proves.
Owner console operates.
Developer console observes.
Invariant core governs.
Audit remembers.
Policy versions protect.
Adapters preserve change.
```

## Intended Use

Use this pack to brief a CLI coding agent, frontend engineer, product designer, or developer maintaining the homestay system.

It is suitable for:

- greenfield build planning
- brownfield migration from F&B template
- owner/admin console architecture
- handover documentation
- implementation prompting
- validation and acceptance testing
