# Frontend Design Philosophy

## Core thesis

```text
A warm, proof-led, media-rich homestay frontend that feels real, private, locally grounded, and quietly premium — with restrained motion, clear booking action, and enough tactile warmth to avoid looking sterile or AI-generated.
```

Short version:

```text
Warm private retreat, proven by real media, guided by calm clarity.
```

## What the frontend must make possible

```yaml
must_make_possible:
  - visitor trusts the property within seconds
  - family can imagine staying there
  - event planner can understand practical usability
  - pool, rooms, parking, layout, and rules are visible
  - booking path is obvious on mobile
  - media feels real and believable
  - style remains coherent even when owner changes photos/content
```

## What it must prevent

```yaml
must_prevent:
  - beautiful but vague homestay brochure
  - luxury fantasy that feels fake
  - over-polished AI-upscaled visual distrust
  - video/motion bloat
  - poor mobile booking path
  - hidden rules and unclear capacity
  - random style edits by future agents
```

## Design energies

```yaml
design_energy_blend:
  warm_intimate:
    role: family stay, pool, rooms, kitchen, evening ambience

  reflective_storytelling:
    role: hero, gallery, day/night/rain media

  premium_restrained:
    role: spacing, typography, package cards, event/corporate sections

  practical_proof:
    role: pax, parking, layout, rules, WhatsApp inquiry
```

## Final doctrine

```text
Color creates comfort.
Typography creates trust.
Photos prove reality.
Motion creates presence.
Fallbacks preserve confidence.
Tokens protect coherence.
```
